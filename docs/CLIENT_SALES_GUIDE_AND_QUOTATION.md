# Client Sales Guide and Quotation

## 1. Price Position

You told the client the project can be delivered for **LKR 160,000**.

For this kind of system, that price is acceptable only if you present it as a **ready-made e-commerce platform customized for the client**, not as a fully custom system built from zero.

Current Sri Lankan market references show:

- Basic business websites often start around LKR 50,000 to 100,000.
- Corporate/business websites commonly sit around LKR 75,000 to 300,000.
- E-commerce websites commonly sit around LKR 150,000 to 700,000+ depending on features.
- Custom web applications with admin dashboards, backend APIs, authentication, reporting, and advanced workflows can go from LKR 400,000 to 2,000,000+.

So **LKR 160,000 is a competitive/discounted price** for this project. It is good for closing the sale, but you must protect the scope.

Recommended wording:

> The standard value of this type of e-commerce system is higher, but because we already have a stable platform foundation, I can offer this implementation for LKR 160,000 with defined customization and setup.

## 2. How To Sell The Project

### Main Selling Angle

Do not sell this as "a website".

Sell it as:

> A complete online store management system with storefront, admin panel, product management, orders, quotations, coupons, reviews, customer accounts, and mobile-optimized performance.

### What To Show First

1. Storefront homepage
2. Mobile shop/product browsing
3. Product detail page
4. Cart and order/quotation flow
5. Customer account area
6. Admin dashboard
7. Product/category/brand management
8. Orders, coupons, reviews, returns/refunds
9. Site settings such as logo, contact details, footer, map, and social links

### What To Say During Demo

Use business language:

- "Customers can browse products smoothly on mobile."
- "Admin can manage products, categories, brands, coupons, and orders without calling a developer."
- "The site supports quotation-based sales and normal order flows."
- "The system can be customized with your logo, colors, contact details, and product categories."
- "It is built with modern technology, so future features can be added."
- "The customer owns the deployed site and admin access after final payment."

Avoid saying too much technical detail unless they ask.

### Questions To Ask The Client

Before final confirmation, ask:

1. What is the business name and logo?
2. How many product categories do you need first?
3. How many products should be uploaded at launch?
4. Do you need online card payment now, or only COD/bank transfer/quotation?
5. Who will manage products and orders?
6. Do you already have a domain?
7. Do you need Sinhala/Tamil language support?
8. Do you need delivery fee rules?
9. Do you need WhatsApp ordering?
10. Do you need monthly support after launch?

## 3. Scope For LKR 160,000

This scope is safe for the price.

### Package Strategy

Sell **LKR 160,000** as the **Essential Launch Package**.

This means the client gets the important features needed to start selling online. The system is built in a way that more features can be added later as paid modules.

Recommended wording:

> This package includes the essential features needed to launch your online store. The platform already supports more advanced modules, so if you need extra features later, we can activate or customize them as add-ons without rebuilding the full system.

Also make it clear that the client can choose what they need:

> You do not have to pay for every feature at the beginning. You can start with the essential package and select only the extra features or pages that your business actually needs. If you have your own custom requirement, we can review it and provide a separate add-on price before implementation.
